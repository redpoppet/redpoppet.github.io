基于主线模型和运维上线的算法工作流

graph TB

    建模模型提交和数仓数据就绪确认 --> 主线模型首次调试
    主线模型首次调试 --> 主线模型输出确认
    主线模型输出确认 --> 调度和联调及测试
    调度和联调及测试 --> 上线和备份及监测
    上线和备份及监测 --> 测试调度和周期调度

组内效率监测指标：

  • [x] 模型代码提交次数（gitlab）

  • [x] 数据校验通过次数 （手动，次数越多，效率越高）

  • [x] 主线模型首次调试通过时间（从开始干活到线上调度通过时间）

  • [x] 模型自动化程度（是否所有步骤都已经脚本化，自动化脚本个数）

  • [x] 模型一次迭代代码修改量（gitlab）
