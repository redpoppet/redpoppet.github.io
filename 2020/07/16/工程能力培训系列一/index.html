工程能力培训系列一

发表于 2020-07-16 | 分类于 Technical Innovation

总体安排

为了提升通识能力，基于公司业务，按照当前情况预计安排三次工程开发通识培训。

  • 第一次：工作答疑系列。主要负责当前工作中工作岗位内容认知，模型流转过程，开发平台体系相关的疑问解答

  • 第二次：基础培训系列。主要讲解计算机主要课程的相关概要

  • 第三次：视野扩展系列。主要讲解一些主流的开发体系和通用做法。

培训大纲：

经过调研，第一次的培训内容答疑问题安排如下：

  1. 我们提交模型之后，到客户看到之前，中间发生了啥？

  2. 什么叫服务？

  3. 什么叫微服务？

  4. 介绍下产品架构，当初为什么做了这样的架构？架构里面的组件为啥选择这些？

  5. 开发的时候用的是开发环境的配置，到了测试环境是怎么自动更新的？

  6. 面向服务的架构，Docker，DevOps这些专业名词都是啥意思？干啥的？

主要内容：

1. 我们提交模型之后，到客户看到之前，中间发生了啥？

  1. 代码合并，解决冲突。
  2. 代码加密，合并代码到加密分支。
  3. 代码测试，多机部署测试（可选）
  4. 性能有无问题，有问题另外处理
  5. 测试通过，代码提测，发送合并请求给技术leader。测试不通过，反馈给算法工程师。
  6. 技术leader接收合并请求，下载各端代码，部署到开发环境，冒烟测试联调无误后，发送各端代码合并请求，发送提测邮件，
  7. 测试接收合并请求，下载各端代码，部署到测试环境，然后开始功能测试，数仓测试，模型测试，联调测试。发送测试结论和测试报告。
  8. 测试通过后，项目经理，业务分析师确认效果无误（可选）。
  9. 测试打包所有的代码到svn，然后告知项目经理测试通过。
  10. 项目经理发送上线申请邮件，申请上线。
  11. 运维接受邮件，去svn下载代码包，执行部署脚本，分别部署各端代码(jenkins), 执行环境安装脚本，执行数仓变更脚本，回复上线邮件
  12. 项目经理进行回归测试，确认整体联调成功，模型输出结果无误，然后邮件回复此次上线成功。
  • 天天说的服务是什么？
    • 有请求；
    • 有响应;
    • 常驻内存等待;
    • 不对外暴露内部处理逻辑。

5. Docker，DevOps，k8s这些专业名词都是啥意思？

  • [x] Docker 可以让开发者打包他们的应用以及依赖包到一个轻量级、可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。

  • [x] DevOps是一组过程、方法与系统的统称，用于促进开发（应用程序/软件工程）、技术运营和质量保障（QA）部门之间的沟通、协作与整合。透过自动化“软件交付”和“架构变更”的流程，来使得构建、测试、发布软件能够更加地快捷、频繁和可靠。

  • [x] k8s 它是容器集群管理系统，是一个开源的平台，可以实现容器集群的自动化部署、自动扩缩容、维护等功能。

  • [x] CDH Cloudera 提供一个可伸缩，稳定，综合的企业级数据管理平台，用于管理快速增长的数据，使用户可以快速部署和管理Hadoop 及相关大数据处理框架，操作、分析企业级数据，并保证数据的安全性。
